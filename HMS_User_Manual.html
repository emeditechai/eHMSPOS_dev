eLuxStay — Complete User Manual Hotel Management System — End-to-End Operating Guide

eLuxStay — Complete User Manual

This is the end-to-end operational guide for the eHMS Hotel Management System. Every page, every button, and every action is documented here with full dependency chains. A new user can follow this manual from Chapter 1 onward and operate the entire system without any prior training.

Version: HMS POS v2.0 May 2026 Audience: Front Desk, Cashier, Manager, Admin
Critical — Read Before Anything Else

⚠ Mandatory Setup Order

🚫
Do NOT skip this! The system has strict parent-child relationships. If you try to create a Booking before setting up Rooms, or create Rooms before creating Floors — the system will show empty dropdowns and errors.

Always follow this exact sequence when setting up for the first time:

1. Branch → 2. Hotel Settings → 3. Users & Roles → 4. GST Slabs → 5. Floors → 6. Room Types → 7. Rooms → 8. Rate Master → 9. Guests → 10. Bookings
💡
For Banquet: Set up Venue Master, Event Types, and Packages BEFORE creating any Banquet Booking.
Chapter 1

⚙ Initial System Setup

Branch Master

Settings›Branch Master

A Branch represents a physical hotel property. Every piece of data in the system (bookings, payments, reports) is tied to a Branch. This is the very first record you must create.

⚠
Must be done first. Without a Branch, you cannot configure Hotel Settings, create Rooms, or create Bookings.

1
Navigate

2
Click "Add New Branch"

A form will open with the following fields.

3

Click Save. The branch now appears in the list. All future logins by staff will be associated with a branch.

GST Slab Master

Settings›GST Slab Master

Defines the tax percentage brackets based on room tariff amount. Indian GST rules mandate 12% for rooms below ₹7,500/night and 18% for above. These slabs auto-apply to bookings.

⚠
Must be set before Rate Master or Bookings. If no GST slab exists, the system cannot calculate tax on invoices.

How to Add a GST Slab

1
Navigate to Settings → GST Slab Master
2
Click "Add"

Enter: Min Amount, Max Amount, GST %. Example: Min=0, Max=7499, GST=12%.

3
Repeat for each bracket

Add a second row for 7500+ at 18%. Slabs must cover all price ranges without gaps.

Cancellation Policies

Settings›Cancellation Policies

Policies define how much the hotel retains when a guest cancels a booking. The system uses this automatically when staff initiates a cancellation — calculating the penalty and computing the refundable amount.

FieldDescription
Policy NameDescriptive name, e.g., "24-Hour Free Cancellation"
Hours Before Check-InWindow in which this rule applies
Deduction TypePercentage or Fixed Amount
Deduction Valuee.g., 50 (means 50% retention if cancelled within 24 hrs)
💡
Multiple policies can be created. The system applies the most appropriate policy based on when cancellation is initiated relative to check-in.

Creating a Booking

Bookings›Bookings → New Booking
Requires Guest Master + Room Master + Rate Master + GST Slabs → New Booking
💡
This is the most important workflow in the system. Follow every step carefully to ensure accurate invoicing.
1
Click "New Booking"

From Bookings → Bookings list, click the New Booking button (top right).

2
Search for Guest

Type the guest's mobile number or name. If found, details auto-fill. If new, click the + Add Guest icon to register them.

3
Set Check-In and Check-Out Dates

Select the arrival and departure dates. The number of nights auto-calculates.

4
Select Room

Choose Room Type first, then the specific Room Number from the dropdown. Only available rooms for the date range are shown.

5
Verify Rate

The system auto-fills the rate from Rate Master. You can override if authorized. GST is computed automatically below the rate field.

6
Add Extra Charges (Optional)

Click "Add Other Charge" to include Laundry, Extra Bed, etc. from Other Charges Master.

7
Record Advance Payment (Optional)

If the guest pays at the time of booking, enter the advance amount and select the payment mode (Cash / Card / UPI). This amount is deducted from the final balance.

8

Choose Channel (Walk-in, Phone, OTA, B2B). This data flows into Channel Performance reports.

9
Save Booking

Click Confirm Booking. A Booking Number is generated (e.g., BK-2026-00123). The booking now appears on the Availability Calendar and in the Bookings list.

✅
After saving, the system can email the booking confirmation to the guest automatically if Mail Configuration is set up.

Chapter 6

📈 Cancellations & Refunds

Cancellation Flow

🚫
Refunds cannot be created manually. The system generates a refund automatically only after a booking is cancelled and the system determines the refundable amount exceeds zero based on the Cancellation Policy.
Booking Confirmed→ Advance Paid→ Cancel Booking→ Policy Applies→ Credit Note Generated→ Refund Queue→ Refund Processed
1
Open the Booking

Go to Bookings → Bookings list. Find the booking. Click "View".

2
Click "Cancel Booking"

A confirmation dialog shows the applicable Cancellation Policy, the retention amount, and the refundable amount.

3
Confirm Cancellation

If the refundable amount > 0, the system automatically generates a Credit Note and adds the booking to the Refunds queue.

4
The booking status changes to "Cancelled"

The room is freed up on the Availability Calendar immediately.

Refunds

Bookings›Refunds
Data from Cancelled Booking→ Credit Note (auto-generated)→ Refunds Page
⚠
This page will only show records if a Booking was cancelled AND the policy allowed a refund. If advance paid was less than or equal to the cancellation penalty, no refund record appears.
1
Go to Bookings → Refunds

You will see pending refund records from cancelled bookings.

2
Click "Process Refund"

Select the outbound payment method (Cash / Bank Transfer / Cheque). Enter the transaction reference number.

3
Mark as Completed

After the actual money is returned to the guest, click Mark Completed. The status changes to "Refunded" and the record appears in the Cancellation & Refund Register report.
