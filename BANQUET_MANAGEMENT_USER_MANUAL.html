Overview

INTRO

The Banquet Management module handles all non-room event bookings — wedding receptions, corporate conferences, birthday parties, social gatherings, and more. It operates as a self-contained system alongside the hotel's room booking module, with its own booking numbers, payment tracking, and reports.

Booking Number Format: Every banquet booking is auto-assigned a unique number in the format BNQ-YYYY-XXXXXX (e.g., BNQ-2026-000001). The counter resets each calendar year per branch.

Phase 1 – Master Setup

ONE-TIME
Complete masters before creating any booking. All pricing, GST rates, venue capacity, and menu details are sourced from masters — nothing is hard-coded. Incorrect master data will directly affect billing.
🏟️

1A — Banquet Venue Master

Banquet → Venue Master

Define each physical space available for events.

  • Venue Code (unique per branch)
  • Venue Name & Type (Banquet / Conference / Lawn / Rooftop / BoardRoom / Exhibition / Other)
  • Capacities: Seated, Buffet, Theater, Cocktail
  • Area (Sq. Ft) & Floor
  • Rate Per Day & Rate Per Half Day
  • GST Rates: CGST %, SGST %, IGST %, SAC Code
  • Amenities: AC, Stage, Projector/LED, Sound System, Parking, In-House Catering, Wi-Fi
🎭

1B — Event Type Master

Banquet → Event Types

Classify the nature of each event for reporting.

  • Event Type Code (unique)
  • Event Type Name (e.g., Wedding, Corporate, Birthday)
  • Description (optional)
  • Icon Class (FontAwesome class for UI display)
Examples: WEDDING, CONFERENCE, BIRTHDAY, ANNIVERSARY, SEMINAR, EXHIBITION
🍽️

1C — Menu Package Master

Banquet → Menu Packages

Define catering menu packages priced per person.

  • Package Code & Name
  • Package Type: VegMenu / NonVegMenu / MixedMenu / SpecialMenu / JainMenu
  • Price Per Pax (₹)
  • Minimum Guarantee Pax
  • GST Rates: CGST %, SGST %, IGST %, SAC Code
  • Includes: Starters, Main Course, Dessert, Beverages, Live Counter
  • Menu Description (printed on documents)
🎤

1D — Addon Services Master

Banquet → Addon Services

Define chargeable add-on services per event.

  • Service Code & Name
  • Service Type: AV Equipment, Decoration, Photography, Videography, Entertainment, Parking, Transportation, Floral, Stationery, Gifts, Security, Other
  • Rate & Rate Type: PerEvent / PerHour / PerPax / PerItem
  • GST Rates: CGST %, SGST %, IGST %, SAC Code
GST Setup Note: For each master, enter CGST % and SGST % for intra-state supplies, and IGST % for inter-state supplies. The system applies IGST automatically when the booking is marked as "Inter-State Supply". Rates from masters are stored in the booking at time of creation and never change if the master is later updated.

🍽️ Step 3 — Menu Package (Optional)

Select a catering package if food is included. Pricing is automatically calculated per pax.

Package
Choose from active menu packages
Package Pax
Number of persons to charge (defaults to Guarantee Pax)
When a package is selected, the system fetches the price per pax, multiplies by the pax count, and computes GST (CGST+SGST or IGST) automatically from master rates. The line breakdown is stored in BanquetBookingPackageLines.
Formula:
Package Base = Price Per Pax × Package Pax
Package GST = Package Base × (CGST % + SGST %) [or IGST % if inter-state]
Package Total = Package Base + Package GST

🎤 Step 4 — Addon Services (Optional)

Add any extra services: decoration, AV equipment, photography, etc.

  1. Click "+ Add Service" to open the addon selector
  2. Choose an addon service from the dropdown — rate and type are auto-filled
  3. Enter the Quantity (units, hours, or pax depending on rate type)
  4. Add optional Notes for that service
  5. The line total (base + GST) is computed instantly and added to the grand total
  6. Repeat for multiple addon services. Each line is stored separately.
Rate Types: PerEvent (flat fee), PerHour (qty = hours), PerPax (qty = persons), PerItem (qty = items). The system uses the addon's configured rate type to determine billing.

💰 Step 5 — Financial Summary & Overrides

Review the auto-calculated totals and apply any overrides.

Venue Amount
Auto-calculated from venue rates + GST
Package Amount
Auto-calculated from package × pax + GST
Addon Amount
Sum of all addon lines + GST
Service Charge
Manual override — additional service charge (₹)
Discount
Manual discount amount (₹)
Round Off
Toggle to auto-round to nearest rupee
Total = (Venue Base + Package Base + Addon Base)
     + Service Charge
     - Discount
     + (Venue GST + Package GST + Addon GST)
     + Round Off
All tax amounts (CGST, SGST, IGST) are broken out individually and stored against the booking. SAC codes from masters are also stored for GST compliance.

💳 Step 6 — Advance Payment (Optional)

Collect an advance/deposit at the time of booking creation.

Reference / Transaction ID

Effect on Booking:
• Advance > 0 → DepositAmount = advance amount
• BalanceAmount = Total − Advance
• If advance = total → PaymentStatus = FullPaid
• If advance < total → PaymentStatus = PartialPaid
• A receipt is auto-generated with a unique receipt number
Save the Booking by clicking "Create Booking". You will be redirected to the Booking Details page showing the booking number and full summary.

Phase 3 – Manage Booking

OPERATIONS

📊 Booking Status Lifecycle

Every booking moves through a defined lifecycle. Each transition is logged in the audit trail.

Inquiry → Tentative → Confirmed → Checked In → Event Complete
Can be cancelled at any point → Cancelled or marked as → No Show
Status Meaning Action Button When to Use
Inquiry Initial booking created — client is enquiring Default on creation When a prospect contacts you but hasn't committed
Tentative Venue held for client, not yet fully confirmed Mark Tentative When venue is reserved but details are being finalised
Confirmed Booking fully confirmed, advance received Confirm After advance payment and final agreement
Checked In Event has started / venue has been handed over Mark Checked In On the day of event, when client arrives and venue is ready
Event Complete Event has concluded successfully Mark Event Complete After event ends, before final billing/settlement
Cancelled Booking is cancelled; refund process initiated Cancel Booking When client cancels or booking cannot proceed
No Show Client did not show up on the event day Mark No Show When event date passes with no client arrival

💳 Add Payment

Collect additional payments after booking is created. Go to the Booking Details page and click "Add Payment".

  1. Open the booking from Banquet → Event Bookings → [Booking Number]
  2. Click the "Add Payment" button on the Details page
  3. Enter the Amount (defaults to outstanding balance)
  4. Select Payment Method
  5. Enter method-specific details:
    • Card: Card Type, Last 4 digits
    • Cheque: Cheque date
    • UPI / Bank Transfer: Reference / Transaction ID
    • Bank: Select bank from list
  6. Optionally enter Discount Amount or Round Off Amount
  7. Add Remarks if needed
  8. Click "Save Payment" — a receipt number is auto-assigned

Supported Payment Methods

💵 Cash 💳 Card (Debit/Credit) 📱 UPI 📜 Cheque 🏦 Bank Transfer
After full payment, PaymentStatus automatically updates to FullPaid. The balance amount shown on the details page is always current.

❌ Cancel Booking

Cancellations are governed by the Cancellation Policy attached to the booking. The refund amount is computed from the policy rules.

  1. Open the booking and click "Cancel Booking"
  2. System displays a Cancellation Preview:
    • Days before event
    • Amount paid so far
    • Refund % (from cancellation policy)
    • Flat deduction (from cancellation policy)
    • Calculated deduction amount
    • Net refund amount
    • Policy name applied
  3. Enter a Cancellation Reason (required)
  4. Optionally enter an Additional Flat Deduction (for penalty overrides)
  5. Click "Confirm Cancellation"
  6. Booking status changes to Cancelled and the refund process is initiated
Important: If no cancellation policy was assigned to the booking, the system will cancel without any refund deduction — the full amount paid is eligible for refund. Always assign a policy at booking time.

Phase 4 – Reports

REPORTING

Navigate to Banquet → Reports to access all five standard reports.

📥 Collection Register

Daily payment collection summary. Shows all payments received for banquet bookings — grouped by day with totals per payment method.

Filters: From Date To Date
Includes: Daily totals, payment method split, booking-level detail, receipt numbers.

🧾 GST Register

Tax ledger for all banquet bookings. Lists each booking with base amount, CGST, SGST, IGST, and total tax per event.

Filters: From Date To Date
Default range: 1st of current month to today. Use for monthly GST filing.

🏟️ Venue Utilization

How each venue is being used. Shows booking count, total revenue, and utilization days per venue for the selected period.

Filters: From Date To Date
Use this to identify peak venues and plan capacity management.

🎭 Event Type Performance

Revenue and booking count breakdown by event type (Wedding, Conference, Birthday, etc.) for the selected year.

Filters: From Date To Date
Default range: 1st January of the current year to today.

⚖️ Outstanding Balance

All bookings with an unpaid balance. Shows booking number, event details, total amount, amount paid, and balance due.

Filters: No date filter – all time
Review this daily to follow up on pending collections, especially for B2B credit bookings.

GST Reference

TAX

How GST is Applied

GST rates are sourced entirely from master data. The system never uses hardcoded rates.

Component Rate Source Intra-State Inter-State
Venue Hire BanquetVenue Master CGST % + SGST % IGST %
Menu Package BanquetPackage Master CGST % + SGST % IGST %
Addon Services BanquetAddonService Master CGST % + SGST % IGST %
Inter-State Toggle: On the booking form (B2B section), toggling "Inter-State Supply" switches all GST calculations to IGST. The IGST amount = CGST + SGST combined. Relevant SAC codes are stored per line for compliance.
GST Snapshot: All GST rates are captured and stored in the booking at the time of creation. If a master GST rate changes later, it does NOT affect existing bookings. This ensures audit compliance.

Quick Reference

CHEAT SHEET

🗺️ End-to-End Action Flow

1. Setup Masters (One-Time)

Create Venues → Event Types → Menu Packages → Addon Services with correct GST rates

2. Check Availability Calendar

Confirm the venue is free on the requested date before taking the booking

3. Create Booking

Banquet → New Booking → Fill all 6 steps → Click Create → Booking gets BNQ number

4. Collect Advance

Either at creation (Step 6) or via Add Payment on Details page

5. Confirm Booking

Click "Confirm" button → Status: Inquiry → Confirmed

6. Day of Event — Check In

Click "Mark Checked In" → Status: Confirmed → CheckedIn

7. Collect Balance Payment

Add Payment → collect outstanding balance before/after event

8. Complete Event

Click "Mark Event Complete" → Status: CheckedIn → EventComplete

9. Verify Reports

Review Collection Register and GST Register for the day's settlements

🔧 Common Scenarios & Solutions

Scenario Action
Venue already booked on that date System blocks save → Choose another venue, time slot, or date
Client wants to change venue Edit booking → Update venue → System recalculates venue cost & checks availability
Client cancels day before event Cancel Booking → Preview shows policy-based deduction → Confirm cancellation
Additional decoration added last minute Edit booking → Add Addon Service (Decoration) → Save → New total calculated
Client pays in installments Create booking with initial advance → Add Payment multiple times as received
Corporate client needs GST invoice Set CustomerType = B2B → Enter Company GSTIN → Mark inter-state if applicable
Need to check which venues are free next Saturday Open Availability Calendar → Navigate to next Saturday → Check unblocked venues
Monthly GST filing Reports → GST Register → Set date range to full month → Export / Print
Follow up on unpaid bookings Reports → Outstanding Balance → Shows all bookings with pending amounts
Guest registered but didn't attend Mark No Show on the booking details page
